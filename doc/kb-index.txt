PLUGIN RELATIONSHIPS                                        *kb-index-relations*

    Understanding how the main plugins connect:

    Mason  →  installs the binaries (LSP servers, DAP adapters)
    lspconfig  →  tells Neovim how to start those LSP servers
    nvim-cmp  →  uses the running LSP to power completion suggestions
    nvim-dap  →  uses the installed DAP adapters to run debug sessions

    In plain terms: if a language feature isn't working, check Mason first
    to confirm the server is installed, then check |kb-lsp| for the mapping.
    If a debug session won't start, check Mason for the DAP adapter, then
    check |kb-debugging|.
